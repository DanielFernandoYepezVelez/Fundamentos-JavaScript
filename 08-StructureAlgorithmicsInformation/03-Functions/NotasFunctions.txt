FUNCIONES =>
Son fragmentos de codigo que escribimos para ejecutar una tarea y no volver a escribir el mismo codigo mas de una vez.

Nos ayudan a modularizar el codigo.

Las funciones deben realizar una sola tarea.

Las funciones siempre deben retornar algo(variables, arrays, objetos, funciones), si no lo hacen, esto no se llama funcion, su definición es procedimiento.

Se deben ver como variables que retornan un valor, pero en este caso son funciones.
-----------------------------------------

SYNTAXIS DE HACE TRES AÑOS =>
function nombreFuncion() {
  //codigo;
}
nombreFuncion();

SYNTAXIS ACTUAL => 
const nombreFuncion = () => {
  //codigo;
}
nombreFuncion();
-----------------------------------------

RETURN =>
Es muy importante recordar que lo que se devuelva en los returns van a ser el valor del la function completa.
Recordar que puede retornar un valor, una variable, un arreglo, un objeto, pero las funciones siempre deben retornar algo, es la mejor practica.
Cuando Estoy Retornando Un objeto la sintaxis del return es la siguiente:
                    return {
                      ObjetoRetornado
                    }
